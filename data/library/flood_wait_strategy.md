# FloodWait — стратегия обработки

## Суть
Telegram ограничивает `ImportChatInviteRequest` скользящим окном ~5 минут.
6-й вызов в окне = FloodWait.

## Правила

### MAX_FLOOD_WAIT = 600 секунд
- Если `wait <= 600` → `asyncio.sleep(wait + 3)` → retry
- Если `wait > 600` → пропустить канал, записать "flood_too_long:{N}"

### Наблюдаемые значения
- 6-й `ImportChatInviteRequest` в окне: ~125с (стабильно)
- 5-й (если неудачный порядок): 3–30с (вариативно)
- `StartBotRequest`: флудвейт редко, ~10–30с если есть

### InviteRequestSentError = УСПЕХ
Для approval-каналов с модерацией `InviteRequestSentError` означает
что заявка отправлена — это УСПЕХ, реф засчитается.
Не считать это ошибкой!

## Параллельность спасает
FloodWait у каждого аккаунта — НЕЗАВИСИМЫЙ.
Аккаунт A ждёт 125с, аккаунт B ждёт 125с — одновременно.
Итоговое время = время самого медленного аккаунта, не сумма.

→ Всегда запускай CONCURRENCY = len(accounts), никогда не батчи по 3.
