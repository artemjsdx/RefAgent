# CTX_LIMIT_180 (FavoriteAPI)

**Ошибка:** `log_code: "CTX_LIMIT_180"` в ответе FavoriteAPI

## Причина
Достигнут лимит контекста 180KB для данного API ключа.

## Процесс управления контекстом
1. После каждого ответа проверяй `context_kb` в ответе
2. При `context_kb > 150` — сжать контекст через memory tag:
   ```
   【⊕write:ctx⊕】
   Краткое саммари разговора на английском (экономит ~60% токенов).
   【⊕/write:ctx⊕】
   ```
3. При `context_kb >= 180` (CTX_LIMIT_180) — сделать сброс:
   - `POST /api/v1/reset` 
   - В первом сообщении после сброса добавить `【⊕load:mem⊕】`
   - Это восстанавливает сохранённые через `write:fav` настройки роли

## Постоянная память (не сбрасывается при reset)
```
【⊕write:fav⊕】
Роль и правила которые должны сохраняться навсегда.
【⊕/write:fav⊕】
```

## В коде
`FavoriteAPIProvider.needs_compression` → True при > 150KB  
`FavoriteAPIProvider.at_limit` → True при >= 180KB
